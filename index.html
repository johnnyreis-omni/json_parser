How to use this tool

A quick guide for building and validating Omnisend personalization tags.

1. Load JSON

Paste a valid event JSON payload. The tool parses it automatically. You can also click JSON Breakdown or press Ctrl/⌘ + Enter.

If JSON is invalid, the error shows the exact line, column, and a caret pointing to the issue.

2. Find fields

Use Quick search to search by key or sample value. Click a result to open the tree directly to that field.

You can also browse the tree with mouse clicks or keyboard arrows.

3. Build tags

Select a field to generate a tag like [[ event.raw.order_number ]]. Use the prefix dropdown to include or remove event..

Add an optional fallback to generate | default: 'value'.

4. Check one field

Use Does this field exist? to test a path like raw.order_number or a full tag like [[ event.raw.order_number ]].

5. Validate tags

Open Validate multiple tags, paste one tag per line, and the tool shows whether each tag is valid, invalid, or empty in the loaded payload.

6. Compare payloads

Click Compare Payloads to paste two events side by side. The tool lists fields only in A, only in B, and shared fields with different values.
